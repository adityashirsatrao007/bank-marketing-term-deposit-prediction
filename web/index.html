Production Recommendation

Ensemble Superiority: The Gradient Boosting model outperforms Logistic Regression across almost all standard offline indicators, notably establishing a stronger ROC-AUC Score. This means GBT is significantly more effective at ranking clients by subscription likelihood without needing high-precision threshold calibrations.

Business Application: If campaign budget is limited, use the Gradient Boosting model to target the top 10% highest-scoring clients. For wider targeting with lower risk of missing potential clients, Logistic Regression remains a highly robust secondary choice due to its slightly higher Recall.

Project Overview & Methodology

Business Case & Problem Statement

Direct marketing campaigns (specifically telemarketing) represent a substantial resource expenditure for banking institutions. The goal of this project is to develop and analyze predictive machine learning classification pipelines to determine whether a client will subscribe to a bank term deposit (labeled yes/no in the variable y).

By leveraging historical campaign records from the UCI Bank Marketing Dataset (45,211 instances, 17 variables), the Aegis pipeline separates high-likelihood candidates from uninterested leads, permitting optimization of marketing resources.

Dataset Features & Schema

  • Demographics: Age, Job, Marital Status, Education Level.
  • Financial Status: Average Yearly Balance, Credit Default records, Housing debt, Personal Loans.
  • Current Campaign: Contact Communication Type (cellular/telephone), Call Day of Month, Call Month of Year, Call Duration (seconds).
  • Historical Campaign: Number of Contacts during campaign, Days elapsed since previous contact (pdays), Number of previous contacts, Previous campaign outcomes (success/failure).

Data Pipeline & Modeling Rules

To avoid label leakage and guarantee production robustness, the pipeline strictly adheres to the mle-workflow standards:

  1. Data Split: The dataset is partitioned into 80% training and 20% test sets using a Stratified Split. This maintains exact positive-to-negative target proportions across folds and prevents training skew on imbalanced targets.
  2. Preprocessing: Numerical values undergo standard scaling. Categorical attributes are encoded using one-hot encoding derived from fixed configuration parameters. This isolates preprocessing logic and eliminates vocabulary mismatch between Python and Javascript serving.
  3. Model Tiers:
    • Logistic Regression: Fit with L2 regularization and standard coordinate descent. It yields highly interpretable coefficient odds ratios.
    • Gradient Boosting: Configured as a decision tree ensemble. It captures complex non-linear correlations and categorical interactions without needing manual feature crossing.

Client-Side Sigmoid Inference Engine

To support high-speed serving and eliminate server runtime costs, the trained model parameters (scale factors, coefficients, and decision tree splitting trees) are compiled into a compact JSON payload. The interactive client simulator on Tab 1 uses a custom in-browser JavaScript engine to evaluate these equations in real-time, matching Python prediction decimals exactly.
